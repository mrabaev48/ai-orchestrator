1. Feature summary
Ввести production-ready persisted `ExecutionPolicyDecision` для всех side-effect действий (commit/push/PR, запись артефактов, внешние API вызовы) с обязательной проверкой policy до выполнения эффекта.

2. Relevant modules
- `packages/core/policy/**` (domain contract и инварианты)
- `packages/application/policy/**` (orchestration gate)
- `packages/state/**` (persist/read policy decisions)
- `packages/execution/**` (enforcement перед side-effect шагами)

3. Existing behavior
Policy-checks частично определены, но нет гарантии, что каждое side-effect действие связано с сохраненным, трассируемым и воспроизводимым policy-решением.

4. Proposed design
- Добавить строго типизированный `ExecutionPolicyDecision`:
  - `decisionId`, `tenantId`, `projectId`, `runId`, `stepId`, `attempt`, `actionType`, `riskLevel`, `decision`, `reasonCodes`, `decidedAt`, `decider`, `inputHash`, `traceId`.
- Реализовать append-only persistence для policy decisions (запрет update/delete бизнес-логикой и storage ACL).
- Ввести обязательный guard: side-effect шаг не может стартовать без валидного persisted decision со статусом `allow`.
- Нормализовать deny/error решения в структурированные ошибки (`POLICY_DENIED`, `POLICY_DECISION_MISSING`, `POLICY_DECISION_STALE`).
- Обеспечить deterministic decision replay по `inputHash` + policy version.

5. Files likely to change
- `packages/core/src/policy/execution-policy-decision.ts`
- `packages/core/src/policy/policy-decision-errors.ts`
- `packages/application/src/policy/policy-gate.ts`
- `packages/state/src/policy/**`
- `packages/execution/src/guards/side-effect-policy-guard.ts`

6. Risks
- Race condition между decision persistence и execution start.
- Ошибки связывания `attempt`/`stepId` приводят к ложным deny/allow.
- Cross-tenant leakage при неверной фильтрации policy records.

7. Test plan
- Unit: валидный allow decision открывает side-effect path.
- Unit: missing decision блокирует side effect с типизированной ошибкой.
- Unit: deny decision корректно останавливает выполнение и пишет evidence ссылку.
- Integration: policy decision сохраняется и однозначно линкуется к run/step/attempt.
- Regression: повторный запуск с тем же inputHash не создает противоречивых решений.

8. Rollout / migration notes
- Additive rollout через feature flag `policyDecisionPersistenceV1`.
- Миграция: legacy запуски помечаются `decisionState=unknown`, без ретроактивного allow.
- Добавить runbook для операционного расследования policy-deny инцидентов.

9. Recommendation
Сделать первым PR в mini-sequence; downstream задачи 1.2+ не должны переходить в ready без завершения этой задачи.
