# Task 12.2 — Enforce shared run-lock DSN for multi-worker

**Priority:** P1

Parent task: `12-introduce-safe-write-pipeline.md`

## Goal
Close the coordination risk by enforcing that all workers in multi-worker mode use one shared lock authority DSN (`WORKFLOW_RUN_LOCK_DSN`).

## Scope
- Add explicit runtime config validation for multi-worker lock requirements.
- Reject startup when lock provider/DSN policy is invalid.
- Keep single-worker fallback behavior explicit and safe.

## Acceptance criteria
- `workerCount > 1` without `WORKFLOW_RUN_LOCK_DSN` fails fast at config load.
- `noop` provider is rejected for multi-worker mode.
- Provider and DSN scheme mismatch is rejected with a deterministic error message.
- Tests cover valid and invalid multi-worker lock configurations.
