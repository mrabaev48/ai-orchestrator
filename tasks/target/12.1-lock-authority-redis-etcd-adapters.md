# Task 12.1 — Add Redis/etcd LockAuthority adapters

**Priority:** P1

Parent task: `12-introduce-safe-write-pipeline.md`

## Goal
Close the risk that only one lock authority backend is practically usable by implementing and hardening dedicated LockAuthority adapters for Redis and etcd.

## Scope
- Add production-safe Redis LockAuthority adapter.
- Add production-safe etcd LockAuthority adapter.
- Keep adapter behavior isolated behind the existing LockAuthority port.
- Keep orchestration/domain flow unchanged except for provider selection.

## Acceptance criteria
- Redis adapter supports acquire/release with ownership token checks.
- etcd adapter supports lease-based acquire/release semantics.
- Adapter failures surface as structured configuration/runtime errors.
- Unit tests cover success and contention/failure scenarios for both providers.
