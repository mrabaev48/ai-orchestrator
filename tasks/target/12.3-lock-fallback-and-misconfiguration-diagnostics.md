# Task 12.3 — Lock fallback and misconfiguration diagnostics

**Priority:** P1

Parent task: `12-introduce-safe-write-pipeline.md`

## Goal
Close operational risk by making fallback behavior and lock misconfiguration diagnostics explicit for operators.

## Scope
- Document NoopLockAuthority as single-node fallback only.
- Add deterministic runtime signals when a run lock cannot be acquired.
- Add actionable error messages for missing lock dependencies/config mismatch.

## Acceptance criteria
- Run cycle returns an explicit idle stop reason when global run lock is unavailable.
- Misconfigured lock provider fails with actionable config/runtime errors.
- Operator-facing docs/tasks clearly state: multi-worker requires shared `WORKFLOW_RUN_LOCK_DSN` across all workers.
- Regression tests cover lock-unavailable path and key config error paths.
